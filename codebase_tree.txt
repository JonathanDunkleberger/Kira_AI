PER-FILE SUMMARIES  (.py)

────────────────────────────────────────────────────────────────────────────
CORE RUNTIME
────────────────────────────────────────────────────────────────────────────

bot.py (4816)
  The central orchestrator. Owns the asyncio event loop, all background-task
  scheduling, the brain worker (input triage via Groq), the dynamic observer /
  boredom loop, the chat batch worker, vision/audio heartbeat wiring, and
  session artifact generation. Holds the central state machine: mode
  (companion/streamer), activity (game/VN/media/general), immersive-vs-observer,
  Carry/Drive mode, pause/mute, and emotion. Also owns prompt assembly
  (_build_scene_block, the response-shape selector, voice guardrails) and
  shutdown_async.

ai_core.py (1330)
  Inference + speech engine. Routes text generation to Claude (Sonnet for live
  chat, Opus for deep moments) or to Groq/local-Llama via inference_router.
  Owns Azure and Fish TTS (with Edge fallback), Whisper STT init, caption
  word-boundary syncing, per-session token accounting, and the emotion-analysis
  triage call. Builds the full system prompt each call (persona + memory + scene
  + audio blocks) with aggressive truncation.

dashboard.py (1928)
  CustomTkinter desktop control panel. 4-column layout (State | Perception |
  Activity | Live Controls) plus a 500ms UI refresh loop and a separate VRAM
  poll loop. Owns every operator toggle: vision, audio mode, loopback STT,
  Carry Mode + drive-agenda text, VN autopilot, Media Watch, mute/pause, TTS
  backend switch, activity GO button, mode toggle, VRAM auto-degrade, and the
  graceful-shutdown window-close handler.

vn_autopilot.py (3232)
  Autonomous visual-novel reader. Phase 1: screenshot → classify
  (DIALOGUE/CHOICE/SAVE/TRANSITION) → OCR → react → advance. Phase 2 adds six
  "watchability" systems: dynamic pacing, dead-chat behavior, theory-building,
  character attachment, narrative-weight detection, and soft-pause for Jonny.
  Owns VN window targeting + input control, speaker attribution, and a heavy
  parallel reaction/theory/aside scheduler. Deeply wired to dashboard + kira_state.

playthrough_memory.py (909)
  Per-game autobiographical record system. Owns playthroughs/<slug>.md
  (rolling summary, opinions, session logs, signature moments) and the
  .checkpoints/<slug>.json crash-recovery files. Regenerates rolling summaries
  every 3 sessions and extracts 0–2 signature moments per session via Claude.

audio_agent.py (762)
  System-audio capture + mood description. MEDIA mode = WASAPI loopback (game
  audio); MUSIC mode = mic. Sends vibe summaries to gpt-audio-mini (not
  transcription) on a heartbeat, with an RMS silence gate and an 80+ entry
  meta-reply filter to suppress confabulation. Also wraps AudD song-ID. Owns
  parallel 16 kHz + native-rate buffers.

control_server.py (672)
  FastAPI + uvicorn backend running inside the bot event loop on 127.0.0.1:8766.
  Serves web_dashboard/index.html, a /state snapshot, a 500ms /ws state push,
  a /vision/thumbnail JPEG, and a POST /cmd/{action} dispatcher mirroring every
  dashboard control. Pure state query + bot-method dispatch (no LLM calls).

loopback_transcriber.py (616)
  Dedicated Whisper (distil-large-v3, ~1.5GB) for rolling MEDIA-mode
  transcription of game/show audio. Owns a 60s rolling segment deque,
  hallucination filtering, a TTS-gate so Kira doesn't transcribe herself, and a
  condensed "dialogue summary". Runs in its own background thread.

memory.py (540)
  ChromaDB vector store wrapper. Owns three persistent collections — turns,
  facts, chatters — with semantic search, deterministic keyword lookups, a 30s
  facts cache, and startup HNSW/SQLite health checks. Pure storage; extraction
  is delegated to memory_extractor.

kira_state.py (506)
  Unified session-intensity system (SessionIntensity enum) shared by VN
  autopilot and the game/general observer. Owns entity attachment / sentiment
  ledger and two background Claude-Sonnet tasks: rolling narrative summary
  (≥60s) and theory formation (≥90s, max 5 open). Bridges intensity to under_load.

stream_logger.py (411)
  Non-blocking session logging. Owns the per-session directory and writes
  transcript.md, events.jsonl, and summary.md via a 30s background flush loop.
  On finish() optionally calls Claude (Opus/Sonnet) for the post-stream summary.
  Bulletproofed: disk errors set a flag and suspend writes rather than crash.

vision_agent.py (359)
  Screen observation for game/VN/media. Owns the vision heartbeat (30s default,
  10s immersive/game), rolling scene summary, on-demand visual Q&A, and "read
  the screen" transcription. Gated by is_active (the operator vision toggle).

media_watch.py (365)
  Separate, isolated vision pipeline for movies/episodes. Captures a frame every
  2.5s into an 8-frame buffer and sends a batch to gpt-4o-mini every 18s to build
  an episode event log (cost ~$0.002/call). Surfaces that log for Q&A and an
  optional reaction callback.

vts_client.py (309)
  VTube Studio WebSocket client (Live2D). Owns connection lifecycle, token
  persistence, hotkey enumeration, and fire-and-forget expression triggering with
  stale-ID re-resolution on model swap. Never raises to caller.

caption_server.py (288)
  Local caption WebSocket server broadcasting JSON frames (caption/clear/cookie/
  chaos) to OBS browser overlays. Owns port-collision retry, per-client handling,
  and silent frame-drop when no overlay is connected. Live-only; no persistence.

identity_manager.py (262)
  Flat-JSON identity store (memory_db/identity.json). Owns Tier-1 permanent
  anchors (Jonny, regulars), Tier-2 activity-scoped characters, the last-30
  session log for temporal continuity, and source-attribution labels for prompts.
  Synchronous, no external APIs.

youtube_bot.py (245)
  YouTube live-chat listener via pytchat. Extracts video IDs from various URL
  forms, polls chat every 2s with exponential-backoff reconnect, pushes messages
  to the input queue. Read-only (no Data-API posting).

twitch_bot.py (218)
  TwitchIO chat + event listener. Parses USERNOTICE for raids/subs/gifts,
  implements !cookies and !sr (song request), and dispatches messages/events via
  callbacks. Reading only — posting lives in chat_poster.

salience_filter.py (198)
  Pure-Python input gatekeeper (no LLM, no I/O). Owns salience tier assignment
  (HIGH/MEDIUM/LOW/DROP), novelty tracking to suppress repeated ambient content,
  low-tier truncation, and staleness notes for visual headers.

cookie_jar.py (194)
  Thread-safe persistent cookie tracking (shared communal jar + per-chatter
  lifetime). Milestone at 50 rolls over and queues Chaos Mode (15-minute directive
  + end lines). Owns counters, milestone flag, and atomic JSON writes.

vts_expression_controller.py (166)
  Maps EmotionalState → VTS hotkey animation. Only fires for "significant" states
  (MOODY, HYPERACTIVE); others are deadpan. Detects model swaps and flushes stale
  mappings. Fire-and-forget safe.

config.py (162)
  Loads all configuration + secrets from .env: model names, API keys, feature
  flags, rate limits, intervals, and paths. Single source of truth for toggles.

chat_poster.py (159)
  Centralized, rate-limited chat poster (global 60s cooldown, 450-char cap),
  hard-gated by ENABLE_CHAT_POSTING (currently disabled). No-ops when disabled.

theme.py (139)
  Dashboard color palette (warm coffee-shop tones, three depth planes + accents).
  Pure constants consumed by dashboard.py.

streamer_overlay.py (109)
  Streamer-mode persona override text. Appended to the system prompt (uncached, so
  it can flip mid-session) to invert companion guidance toward react-first,
  clippable behavior.

inference_router.py (98)
  Routes synchronous llama-style completions to Groq cloud or local llama_cpp per
  INFERENCE_BACKEND. Returns a llama_cpp-shaped dict. Fallback strategies:
  false / true / lazy_load.

music_tools.py (93)
  mpv + yt-dlp music player. play_kira_song() streams the first YouTube result to
  an mpv subprocess; get_now_playing()/skip_song() manage it. Needs mpv.exe.

groq_client.py (92)
  Thin Groq SDK wrapper. Synchronous chat_completion() returning a llama-shaped
  dict, default 5s timeout (triage must stay fast). Maps Groq errors to one
  GroqInferenceError.

memory_extractor.py (90)
  Async Claude-Sonnet extractor turning conversation turns into durable facts
  (subject/predicate/object or natural-language). High-confidence filter, rejects
  questions/temporary states. Delegates persistence to memory.py.

web_search.py (36)
  Google Custom Search wrapper. Returns up to 3 trimmed snippets; async variant
  wraps it in an executor. Graceful when keys are missing.

twitch_tools.py (57)
  Twitch Helix poll API wrapper. Creates a poll via /helix/polls; fetches
  BROADCASTER_ID dynamically if missing. Gated by ENABLE_TWITCH_POLLS.

prompt_rules.py (22)
  Constants for tool/format tags ([POLL], [SONG], [PREDICT]) and brevity guidance.

prompt_loader.py (9)
  Loads personality.txt into KIRA_PERSONALITY.

persona.py (9)
  EmotionalState enum (HAPPY/MOODY/SASSY/EMOTIONAL) — the canonical emotion set.

personality_file.py (2)
  Re-export shim: `from personality_file import KIRA_PERSONALITY`.


────────────────────────────────────────────────────────────────────────────
ONE-OFF / OFFLINE SCRIPTS  (not part of the live runtime)
────────────────────────────────────────────────────────────────────────────

backfill_lore.py (309)
  Standalone tool: scans logs/streams/ transcripts (and PENDING_*.json survivors)
  and regenerates missing lore/clip entries via Claude. Manual, idempotent.

repair_memory_db.py (192)
  One-time ChromaDB index-repair tool (backs up, reads, deletes, re-adds to force
  HNSW rebuild). Requires exclusive DB access (stop dashboard first).

_diag_memory.py (82)
  Read-only memory diagnostic: inspects a temp copy of chroma.sqlite3 and verifies
  retrieval paths. Underscore prefix marks it as a dev utility.

backfill_signatures.py (69)
  One-time backfill of the Signature Moments section in playthrough files, using a
  minimal Anthropic stub to avoid importing heavy ai_core deps.


================================================================================
BEHAVIOR-SHAPING CONFIG / DATA FILES
================================================================================

personality.txt (~13KB)
  Core persona body injected as the cached system-prompt base. Defines Kira's
  voice, brevity rules, and identity. Loaded via prompt_loader/personality_file.

streamer_overlay.py
  Persona override text for streamer mode (see above) — behavior-shaping, not data.

config.py / .env / .env.example
  All feature flags, model selection, intervals, and rate limits. .env holds
  secrets (keys, OAuth); .env.example is the public template.

cookie_data.json
  Persistent cookie counters: shared total (x/50), milestone count, per-chatter
  lifetime tallies. Drives the cookie-jar milestone + chaos-mode trigger.

lore/general.md, lore/<slug>.md
  Session-end "lore" written by Claude. Loaded into the startup brief and scene
  context. general.md is the largest behavior input (~24KB).

lore/general_opinions.md
  Standing opinions + persistent "running bits" reloaded at startup so callbacks
  survive across sessions.

playthroughs/<slug>.md
  Per-game rolling summary + opinions + signature moments fed into scene context
  during that activity. Bleeds into every response mode for the active game.

playthroughs/.checkpoints/<slug>.json
  Crash-recovery snapshots of in-progress reactions/chat-moments; replayed if a
  session didn't close cleanly.

clips/<date>_<activity>.md
  Clip-candidate markdown written at session end (curation aid; not re-read at
  runtime).

memory_db/identity.json
  Tier-1/Tier-2 entity anchors + the last-30 session log. Directly shapes who Kira
  thinks is speaking and temporal continuity.

memory_db/chroma.sqlite3 + <uuid>/
  ChromaDB metadata (SQLite) + three HNSW vector collections (turns/facts/
  chatters). The semantic long-term memory substrate.

caption_overlay/ (index.html, caption.js, style.css)
  OBS browser source that renders live captions broadcast by caption_server.

cookie_jar_overlay/cookie_jar_overlay.html
  OBS overlay rendering the cookie counter / chaos-mode state.

web_dashboard/index.html (~49KB)
  Browser-based dashboard served by control_server (parallel to the Tkinter one).

theme.py
  Dashboard palette constants (UI only).

docs/ (autopilot specs, playthrough-memory design, code-review-notes,
  kira-architecture.svg)
  Design documentation; not loaded at runtime.
