# Playthrough Memory — Design Doc
**Status: IMPLEMENTED** (see `playthrough_memory.py`)

## Goal
Give Kira persistent, autobiographical records of every game she plays — so experiences bleed into all future conversations as genuine lived memory, not data.

---

## Core Design Principles

**Principle 1 — Global retrieval scope (NOT siloed):**
Playthrough context is injected into every conversation mode: voice, chat batch, observer. A stranger asking in chat "have you played Clannad?" gets a real answer referencing actual session memories, even mid-session of a different game. Easy to get right at the foundation; painful to retrofit.

**Principle 2 — Option A immersion (lived experience framing):**
"I played Clannad for 88 hours and it wrecked me" — not "I have data indicating completion of Clannad." Kira references playthroughs the way a person remembers experiences: as real things that happened to her, with emotional residue.

---

## File Structure

One `.md` file per game, stored in `playthroughs/<slug>.md`:

```
# Playthrough: Clannad

## Rolling Summary
150-200 words in Kira's first-person voice. Full arc, specific character names,
emotional trajectory, memorable chat moments. Where things stand now.
Generated by Claude after every 3rd session from all session entries.

## Opinions & Evolving Takes
Her current hot take. Favorite character. What surprised her. Current rating.
(Updated manually or via future automation.)

## Session Log

### Session 1 — 2026-05-18 (~142 min)

**Story:** [2-4 sentences. Plot beats, character moments, revelations.]

**Kira's reactions:** [1-3 sentences. First person. What hit, what got her.]

**Chat moments:** [What chat did. Or: quiet session.]

**Opinion shift:** [Did her take change? Or: holding steady.]

### Session 2 — ...
```

## Architecture

**`PlaythroughMemory` class** (`playthrough_memory.py`):

| Method | Purpose |
|--------|---------|
| `load_for_game(name)` | Slugify → load/create file → reset session accumulators |
| `get_context_for_prompt()` | Returns manifest + current summary + opinions for injection |
| `tag_reaction(text)` | Records autopilot reactions spoken this session |
| `tag_chat_moment(user, msg, response)` | Records notable chat exchanges in VN/game mode |
| `append_session_entry(...)` | Claude-generates structured entry and appends it at session end |
| `_regenerate_rolling_summary(...)` | Every N sessions: re-reads all entries, writes 150-200w summary |
| `_rebuild_manifest()` | Scans `playthroughs/` → compact one-liner per game for global inject |

**Rolling summary regeneration:** Every 3 sessions (one Claude Sonnet call). Reads all session entries, writes a first-person arc summary. Keeps the "fast load" context fresh without reading the full file every time.

**Games manifest:** Always-on, one line per game: `- Clannad (12 sessions) — I spent over 100 hours...`. Injected globally so Kira always knows what she's played.

## Integration Points in `bot.py`

| Integration | Where | What |
|-------------|-------|------|
| Init | `_main_loop()` | `PlaythroughMemory(ai_core=self.ai_core)` after VNAutopilot setup |
| Activity change (voice) | `brain_worker()` | `load_for_game(detected)` on VN/GAME activity |
| Activity change (dashboard) | `dashboard._set_activity()` | `load_for_game(text)` on VN/GAME type |
| Tag reactions | `_autopilot_speak()` | `tag_reaction(cleaned)` after TTS fires |
| Tag chat moments | `_respond_to_chat_batch()` | `tag_chat_moment(...)` when in VN/game mode |
| Session entry | `_write_session_artifacts()` | `append_session_entry(...)` before artifacts-written flag |
| Voice prompt inject | `process_and_respond()` | `get_context_for_prompt()` appended to `chat_system` |
| Chat prompt inject | `_respond_to_chat_batch()` | `get_context_for_prompt()` appended to `session_context_block` |

## What This Does NOT Do (Future Work)
- Per-character attachment tracking (within-session emotional state — Phase 2 System 4)
- Theory/prediction storage across sessions (Phase 2 System 3 + cross-session persistence)
- Auto-update of Opinions block (currently requires manual edit or future automation)
- Mid-session game-switch artifact: if game changes mid-session, only the final game gets the session entry. Edge case, low priority for Phase 1/2.
